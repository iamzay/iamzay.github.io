前端跨域问题总结

发表于 2017-09-20 | 分类于 Web ， 浏览器 |

起因

现代浏览器大多遵循同源策略，即不允许一个页面的脚本访问其他页面的数据。实现跨域一般有下面这些方法：

  • JSONP
  • document.domain
  • window.name
  • window.postMessage
  • websocket
  • 服务器设置

方案

JSONP

html中的一些标签，如img,script等带有src属性，在src中我们可以填上任何的域名而不会收到同源策略的影响。利用这点，可以动态的修改这些标签来达到跨域的效果。
JSONP的步骤大致是：

  • 使用JS动态创建一个script标签，设置其src属性为要跨域访问的服务器url，url的末尾加上类似”callback=funcName”的参数
  • funcName是全局作用域下的一个函数名
  • 将script标签append到dom树中，这样会使浏览器发送一个get请求
  • 服务器接受请求，将funcName从url中取出，并动态生成一个JS脚本（这个JS脚本会使用data去调用funcName，data使服务器端生成的，也是浏览器端所需要的）
  • 浏览器接受返回的JS脚本并执行之
