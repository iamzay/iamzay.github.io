CSS权威指南第三版总结

发表于 2017-07-15 | 分类于 Web ， CSS |

第7章 基本视觉格式化

以下两个章节的讨论都只针对块级元素

水平格式化

  • 7个水平格式化属性和必须是父元素的宽度
  • 这7个属性中，只有宽度和左、右外边距可以设置成auto，其他必须设置成特定的值或默认为0（border-width默认不是0）
  • 若这三个属性中只有一个设置为auto，则其宽度用来弥补与包含块内容宽度的差距
  • 若这三个属性都不为auto，则margin-rignt自动为auto
  • 如果两个外边距都设置为auto，width为特定值，则外边距相等，使元素水平居中
  • 若三个属性都设置为auto，则等同于默认情况，即两个外边距为0，width为auto
  • 替换块元素（图片）只有一个例外，如果width为auto，则元素宽度是内容的固有宽度

垂直格式化

  • 若高度设为固定值，且小于内容高度，高度并不会随之增加，此时可能出现滚动条，这取决于overflow属性
  • 垂直格式化也有7个属性，其和等于包含块的height
  • 这7个属性中只有height和margin可以设置为auto，其他必须为特定值或者默认为0（border可能为默认为medium）
  • 正常流中若将一个块元素的margin-top或margin-bottom设置为auto，它会自动计算为0，所以不能容易地实现垂直居中
  • 若包含块的高度为auto，则百分数高度会重置为auto
  • 包含块的高度为auto：
    • 若包含块没有上下padding或者border，则包含块的height等于内部元素上下外边框之间的距离，此时子元素的外边距可能超出包含块
    • 若包含块有上下padding或者border，则包含块的height等于内部元素上下外边距之间的距离，此时子元素不会超出包含块
  • 垂直外边距会合并，等于最大的那个外边距

行内元素

  • font-size的确定了各个em框的高度
  • 行间距只应用于非替换元素
  • 内容区：
    • 对于非替换元素，内容区由各字符的em框一起构成
    • 对于替换元素，内容区是元素的固有高度加上可能由的各边距、边框
  • 行内框：
    • 非替换元素，为内容区加上行间距，即行高
    • 替换元素，等于内容区的高度
  • 行框：包含行内框的最高点和最低点
  • 非替换元素的内边距、外边距和边框不会对生成的框产生垂直效果
  • 行内框根据其vertical-align属性值在行中垂直对齐
  • 对于块级元素中的匿名文本，可以想象成有一个虚拟的tag保住了它
  • width和text-align不能应用于行内元素

行内非替换元素

  • 行内框高度的确定：内容区加上上下行间距，这个行间距由(line-hight-font-size)得到，所以可能为负，此时行内框的高度小于内容框，文本可能和其他行的内容发生重叠
  • vertical-align也可用于table-cell元素，top,bottom和middle最为常用

  • 设置line-height最好的方法是使用一个原始数字值，这个数会成为缩放因子并继承下去

  • 外边距不会应用到行内非替换元素的顶端和底端

行内替换元素

  • 行内替换元素的各边距和边框影响其行内框的高度

第10章 浮动和定位

浮动

  • 浮动元素会生成一个块级框，而不论这个元素本身是什么
  • 浮动元素的包含块是其最近的块级元素
  • 如果一个浮动元素在两个合并的外边距之间，放置这个浮动元素时就好像在两个元素之间有一个块级元素
  • 浮动元素会延伸，从而包含其所有后代浮动元素
