React中的生命周期函数

发表于 2017-08-08 | 分类于 Web ， Javascript |

生命周期

在创建一个组件实例并将其插入到dom树中这个阶段会调用下面的生命周期函数:

  • constructor
  • componentWillMount
  • render
  • componentDidMount

当组件重新渲染时调用如下的生命周期函数:

  • componentWillReceiveProps
  • shouldComponentUpdate
  • componentWillUpdate
  • render
  • componentDidUpdate

当组件被从dom树移除时调用:

  • componentWillUnmount

Mounting函数

constructor

一般在这里初始化state或者bind方法，如不进行这些操作，可不定义此函数，若进行，则必须在开头加上super(props)，否则this.props未定义。

componentWillMount

因为这是render之前的生命周期函数，所以在此调用setState不会导致重新渲染。

render

这应是一个纯函数。

componentDidMount

在这里setState会导致重新渲染，可以在此开始网络请求。

Updating函数

componentWillReceiveProps

一般当props改变时会调用这个函数，但调用这个函数不代表props一定变化了。

shouldComponentUpdate

这个函数默认返回true，即每次都会调用后面的render等生命周期函数。如果返回false，则不会调用componentWillUpdate,render,componentDidUpdate（未来可能会变，即返回false也不会阻止后面的函数调用）。
可以让组件继承React.PureComponent获得一个shouldComponentUpdate的实现。

componentWillUpdate

无法在这个函数中调用setState。如果需要根据props的变化调用setState，使用componentWillReceiveProps。

componentDidUpdate

这个函数类似componentDidMount。

Unmounting函数

componentWillUnmount

执行组件移除时的收尾工作，比如关闭timer，取消网络请求等。

其他

setState

这个函数几乎总是导致组件的重新渲染，除非shouldComponentUpdate返回false。
setState函数不会立即执行，所以当你调用这个函数后立即引用this.state得到的很可能不是更新后的状态，在这种情况下为setState函数提供一个回调函数:

forceUpdate

这个函数会导致render()被调用，并且调用子组件的正常生命周期函数。
